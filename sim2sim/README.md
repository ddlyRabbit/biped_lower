# sim2sim — MuJoCo Simulation Tools

MuJoCo-based tools for sim-to-sim validation, trajectory visualization, and ONNX policy testing. All scripts use the light URDF converted to MJCF format.

## MJCF Models

| File | Actuators | Use Case |
|---|---|---|
| `mjcf/sim2sim/robot.mjcf` | Torque motors (`<motor>`) | PD control in Python (`play_mujoco.py`, `stand_mujoco.py`) |
| `mjcf/sim2sim/robot_pos.mjcf` | Position actuators (`<position>` with `kp`/`kv`) | Physics-based trajectory playback (`render_zmp_trajectory_mujoco.py`) |

### Gains in `robot_pos.mjcf` (matches training config)

| Joint Group | Kp | Kv | Force Limit |
|---|---|---|---|
| Hip pitch/roll | 180 | 6.5 | ±100 / ±50 Nm |
| Hip yaw | 180 | 3.0 | ±50 Nm |
| Knee | 180 | 6.5 | ±100 Nm |
| Foot pitch/roll | 30 | 0.3 | ±30 Nm |

## Scripts

### `build_mjcf.py` — Convert URDF to MJCF

Converts the biped URDF (light or heavy) to MuJoCo MJCF format.

```bash
python sim2sim/build_mjcf.py --urdf light
python sim2sim/build_mjcf.py --urdf heavy
```

### `stand_mujoco.py` — Stand at Default Pose

Stands the robot at default joint positions using PD control in Python with torque actuators.

```bash
python sim2sim/stand_mujoco.py                  # interactive viewer
python sim2sim/stand_mujoco.py --headless        # no GUI
python sim2sim/stand_mujoco.py --duration 5      # 5 seconds
```

### `play_mujoco.py` — Run ONNX Policy

Runs a trained ONNX student policy in MuJoCo with PD control. Observation spec (45d) matches `biped_env_cfg.py` / `obs_builder.py`.

```bash
python sim2sim/play_mujoco.py --checkpoint deploy/student_flat.onnx
python sim2sim/play_mujoco.py --checkpoint deploy/student_flat.onnx --headless --duration 10
python sim2sim/play_mujoco.py --checkpoint deploy/student_flat.onnx --cmd_vx 0.3
```

### `play_traj_mujoco.py` — Kinematic Trajectory Playback

Plays a CSV joint trajectory as pure forward kinematics (no physics). Sets `qpos` directly and calls `mj_forward()`. Auto-detects dt from the CSV timestamps.

```bash
# Interactive viewer
python sim2sim/play_traj_mujoco.py --csv deploy/biped_ws/src/biped_bringup/config/trajectory.csv

# Save video (headless)
xvfb-run -a python sim2sim/play_traj_mujoco.py \
    --csv deploy/biped_ws/src/biped_bringup/config/trajectory.csv \
    --headless --video /tmp/traj_kinematic.mp4

# Custom MJCF and speed
python sim2sim/play_traj_mujoco.py --csv trajectory.csv --mjcf mjcf/sim2sim/robot_pos.mjcf --speed 2.0 --loop
```

### `play_csv_physics_mujoco.py` — ZMP Trajectory Physics Playback

Renders a CSV joint trajectory using full MuJoCo physics (gravity, contacts, friction). 
Unlike `play_traj_mujoco.py` which teleports kinematic links, this script computes PD torques at 2000Hz (using hardware Kp/Kd parameters) to trace the ZMP trajectory dynamically.

```bash
# Render headless (auto-skips the 8.5s idle phase in ZMP trajectories)
xvfb-run -a python sim2sim/play_csv_physics_mujoco.py \
    --csv deploy/biped_ws/src/biped_bringup/config/trajectory.csv \
    --video trajectory_physics.mp4
```

**Key settings:**
- Skips first 8.5 seconds of simulation instantly to bypass static holding phases.
- Renders the next 5.0 seconds at 50fps.
- Applies Kp=150 to ankle pitch/roll to maintain contact rigidity during the ZMP test.

**CSV format:**
- Row 0: timestamps (seconds)
- Rows 1–12: joint angles (rad) in order: `L_hip_pitch, L_hip_roll, L_hip_yaw, L_knee, L_foot_pitch, L_foot_roll, R_hip_pitch, R_hip_roll, R_hip_yaw, R_knee, R_foot_pitch, R_foot_roll`

### `render_zmp_trajectory_mujoco.py` — Physics-Based Trajectory Render

Renders a CSV trajectory with full MuJoCo physics (gravity, contacts, free base). Uses `robot_pos.mjcf` with position-controlled actuators and the updated ankle gains (Kp=30, Kv=0.3). Outputs MP4 video with a tracking camera.

```bash
# Render (headless, requires xvfb on servers)
xvfb-run -a python sim2sim/render_zmp_trajectory_mujoco.py --csv /tmp/trajectory.csv

# Output: /tmp/zmp_playback.mp4
```

**Key settings:**
- Init height: 0.78m
- Camera: side-tracking, distance 1.5m, elevation -15°
- Physics: full gravity, free floating base
- Subsample: 4× (100Hz sim → 25fps video)
- ~20 min render time on CPU for 26s trajectory

## Dependencies

```bash
pip install mujoco numpy
pip install imageio imageio-ffmpeg   # for play_traj_mujoco.py video export
pip install mediapy                   # for render_zmp_trajectory_mujoco.py video export
pip install onnxruntime               # for play_mujoco.py policy inference
```

For headless rendering on servers without a display:
```bash
sudo apt-get install xvfb libosmesa6-dev
xvfb-run -a python <script>
```

## Generating ZMP Trajectories

The CSV trajectory files are generated by `deploy/scripts/generate_zmp_trajectory.py` which reads `deploy/biped_ws/src/biped_bringup/config/zmp_config.yaml` and uses pinocchio IK to compute joint angles from the ZMP walking pattern. See the deploy README for details.

```bash
# On Pi (has pinocchio installed):
cd ~/biped_lower
python deploy/scripts/generate_zmp_trajectory.py \
    --config ../biped_ws/src/biped_bringup/config/zmp_config.yaml \
    --output ../biped_ws/src/biped_bringup/config/trajectory.csv
```
